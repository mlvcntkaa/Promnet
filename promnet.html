TUGAS RANGKUMAN

Version Control System (VCS)

• Sebuah system yang menyimpan rekamam perubahan pada source code

• Memungkinkan bekerja berkolaborasi dengan lebih baik

• Mengetahui siapa yang melakukan dan kapan sebuah perubahan terjadi

• Memungkinkan kita untuk Kembali ke keadaan sebelum perubahan

GIT

Sebuah vcs terdistibusi untuk mengelola perubahan file di dalam folder (repository/repo) . Git adalah sebuah piranti perangkat lunak berbasis Version Control System, dengan git kita mampu mengelola sebuah proyek atau berkas secara terdistribusi, karena perubahan yang tercatat oleh git disimpan didalam database git dan database git tersebut tidak hanya berada pada satu tempat, namun terdapat di berbagai tempat. Artinya bahwa semua orang yang berkontribusi pada proyek tersebut bisa menyimpan database perubahannya.

Fungsi git

1. Kolaboratif

Dengan menggunakan GIT, dapat bekerja bersama programmer lain dalam membuat atau mengembangkan sistem, setiap programmer akan bekerja di dalam branch-nya masing-masing. Perubahan yang terjadi di dalam code program juga akan lebih mudah dilacak. Menariknya, jika code program selesai dibuat, maka setiap code program dari masing-masing programmer dapat disatukan dengan mudah.

2. Open source

GIT adalah software yang bersifat open source. Hal ini berarti Anda dapat menggunakan aplikasi GIT secara gratis, tanpa biaya tambahan apapun.

3. Membantu organisir

Dengan menggunakan GIT Anda dapat membuat folder project berdasarkan versinya, misal project v1, v2 dan seterusnya. Kemudian,akan memiliki 1 project dengan menggunakan database khusus yang berisi semua file versi projek yang dimiliki.

Istilah pada GIT :

• Repository /repo : folder project atau database yang menyimpan riwayat perubahan.

• Commit : rekaman yang disimpan di repo

• Hash : penanda unik pada sebuah commit

• Checkout : berpindah sebuah commit

• Branch : cabang bebas dari sebuah commit.Biasanya setiap programmer yang tergabung dalam mengerjakan suatu projek, bekerja pada branch-nya masing-masing.

• Merge : menggabungkan branch

• Remote : sumber yang memiliki repo

• Clone : mengambil repo dari remote

• Push : mengirimkan perubahan dari repo local ke repo remote

• Pull : mengambil perubahan dari reporemote ke repo local

• Head:ujung Branch yang mana merupakan commit terbaru
